# 夜间运行需求

## 夜间运行目标
- 每晚定时在非高峰时段完成全量/增量数据处理，减少对在线业务的资源争用
- 保证关键报表与监控结果在次日开始前生成，支持业务决策
- 在夜间窗口完成自动化校验与异常告警，降低人工介入

## 数据来源
- 生产数据库定时快照或增量 binlog/CDC 流
- 对象存储中的原始日志/事件文件（按日期分区）
- 第三方 API 数据拉取（需限频与重试策略）
- 配置中心提供的任务参数与白名单/黑名单

## 审计/合规要求
- 全链路操作日志留存（含任务参数、执行人/执行主体、时间戳）并可追溯
- 数据访问遵循最小权限原则，访问密钥需加密存放并定期轮转
- 任务输出与中间结果按合规要求设置保存周期与访问控制，敏感字段需脱敏/加密
- 失败与异常需记录原因并形成审计事件，支持事后稽核

## 成功判定指标
- 任务整体成功率 ≥ 99.5%，且关键路径无超时
- 次日 06:00 前所有报表/成果文件生成完毕并上传成功
- 数据校验通过率 ≥ 99%，且无 P0/P1 级校验错误
- 异常告警闭环率 ≥ 95%（含自动重试或人工确认）
- 任务执行成本在预算阈值内（如计算/存储费用不超出月度配额）
